## Overview

The MRE Signal Flow attribution system requires comprehensive performance metrics that connect signal generation to P&L impact across 676 tickers and 5 strategies. These metrics enable traders to evaluate strategy effectiveness, optimize allocation, and identify the highest-value signal sources for portfolio performance.

## Core Principle: Signal-to-Outcome Attribution

Every metric traces back to this fundamental relationship:
```
Signal → Price Movement → Portfolio Impact → Performance Attribution
```

The attribution system captures this chain with temporal precision, enabling both real-time monitoring and historical analysis across multiple timeframes.

## Primary Attribution Metrics

### 1. Win Rate Analysis

#### Basic Win Rate
**Definition:** Percentage of signals that achieve their intended outcome within a specified time window.

**Calculation:**
```python
def calculate_win_rate(signals: List[Signal], outcomes: List[Outcome]) -> float:
    """
    Calculate win rate for a strategy-ticker pair over a timeframe.
    
    Args:
        signals: List of signals with type (BUY/SELL/HOLD), timestamp, price
        outcomes: List of actual price movements following each signal
    
    Returns:
        Win rate as percentage (0.0-1.0)
    """
    successful_signals = 0
    total_signals = len(signals)
    
    for signal, outcome in zip(signals, outcomes):
        if signal.type == 'BUY' and outcome.price_change > 0:
            successful_signals += 1
        elif signal.type == 'SELL' and outcome.price_change < 0:
            successful_signals += 1
        elif signal.type == 'HOLD' and abs(outcome.price_change) < 0.02:  # <2% movement
            successful_signals += 1
    
    return successful_signals / total_signals if total_signals > 0 else 0.0
```

#### Time-Decay Adjusted Win Rate
**Purpose:** Weight recent signals more heavily than historical ones.

**Formula:**
```
WR_adjusted = Σ(signal_success_i * decay_factor_i) / Σ(decay_factor_i)

Where decay_factor_i = e^(-λ * days_since_signal_i)
λ = decay constant (default: 0.01 for gradual decay)
```

**Implementation:**
```python
def calculate_time_adjusted_win_rate(
    signals: List[Signal], 
    outcomes: List[Outcome],
    decay_lambda: float = 0.01
) -> float:
    total_weighted_success = 0.0
    total_weight = 0.0
    current_time = datetime.now()
    
    for signal, outcome in zip(signals, outcomes):
        days_since = (current_time - signal.timestamp).days
        weight = math.exp(-decay_lambda * days_since)
        
        success = is_signal_successful(signal, outcome)
        total_weighted_success += success * weight
        total_weight += weight
    
    return total_weighted_success / total_weight if total_weight > 0 else 0.0
```

### 2. Return Metrics

#### Average Return Per Signal
**Definition:** Mean percentage return generated by following each signal.

**Calculation Categories:**

1. **Immediate Return (1-day):** Price movement 24 hours after signal
2. **Short-term Return (1-week):** Price movement 7 days after signal  
3. **Medium-term Return (1-month):** Price movement 30 days after signal

```python
def calculate_average_return_per_signal(
    signals: List[Signal],
    price_history: Dict[str, List[PricePoint]],
    timeframe_days: int = 7
) -> SignalReturnMetrics:
    """
    Calculate return metrics for signals over specified timeframe.
    """
    returns = []
    
    for signal in signals:
        entry_price = signal.price_at_signal
        
        # Find price after timeframe
        future_date = signal.timestamp + timedelta(days=timeframe_days)
        exit_price = get_price_at_date(price_history[signal.symbol], future_date)
        
        if exit_price is None:
            continue  # Skip if no price data available
        
        # Calculate return based on signal direction
        if signal.type == 'BUY':
            return_pct = (exit_price - entry_price) / entry_price
        elif signal.type == 'SELL':
            return_pct = (entry_price - exit_price) / entry_price
        else:  # HOLD
            return_pct = 0  # No position taken
        
        returns.append(return_pct)
    
    return SignalReturnMetrics(
        average_return=statistics.mean(returns) if returns else 0.0,
        median_return=statistics.median(returns) if returns else 0.0,
        max_gain=max(returns) if returns else 0.0,
        max_loss=min(returns) if returns else 0.0,
        return_volatility=statistics.stdev(returns) if len(returns) > 1 else 0.0,
        sample_size=len(returns)
    )
```

#### Risk-Adjusted Return (Signal Sharpe)
**Purpose:** Measure return per unit of risk for each strategy-ticker combination.

**Formula:**
```
Signal_Sharpe = (Avg_Return - Risk_Free_Rate) / Return_Volatility

Where:
- Avg_Return: Mean return per signal over timeframe
- Risk_Free_Rate: 3-month Treasury rate (annualized, then converted to signal timeframe)
- Return_Volatility: Standard deviation of returns per signal
```

### 3. Drawdown Analysis

#### Maximum Drawdown Period
**Definition:** Longest consecutive period of negative performance from a peak.

**Calculation:**
```python
def calculate_max_drawdown_metrics(
    signals: List[Signal],
    cumulative_returns: List[float]
) -> DrawdownMetrics:
    """
    Calculate comprehensive drawdown statistics.
    
    Args:
        signals: Chronologically ordered signals
        cumulative_returns: Running cumulative return after each signal
    """
    peak_value = cumulative_returns[0]
    max_drawdown = 0.0
    current_drawdown_start = None
    max_drawdown_start = None
    max_drawdown_end = None
    max_drawdown_duration = 0
    
    for i, value in enumerate(cumulative_returns):
        # Update peak if new high
        if value > peak_value:
            peak_value = value
            if current_drawdown_start is not None:
                # Drawdown period ended
                current_drawdown_start = None
        else:
            # In drawdown
            current_drawdown = (peak_value - value) / peak_value
            
            if current_drawdown_start is None:
                current_drawdown_start = i
            
            # Update maximum drawdown if worse
            if current_drawdown > max_drawdown:
                max_drawdown = current_drawdown
                max_drawdown_start = current_drawdown_start
                max_drawdown_end = i
                max_drawdown_duration = i - current_drawdown_start
    
    return DrawdownMetrics(
        max_drawdown_pct=max_drawdown,
        max_drawdown_duration_signals=max_drawdown_duration,
        max_drawdown_start_date=signals[max_drawdown_start].timestamp if max_drawdown_start else None,
        max_drawdown_end_date=signals[max_drawdown_end].timestamp if max_drawdown_end else None,
        recovery_time_signals=calculate_recovery_time(cumulative_returns, max_drawdown_end) if max_drawdown_end else None
    )
```

#### Drawdown Frequency
**Purpose:** How often does the strategy experience significant losses?

**Metrics:**
- **Drawdown Events:** Count of periods where cumulative return fell >5% from peak
- **Average Drawdown:** Mean percentage loss during drawdown periods
- **Recovery Efficiency:** Average time to recover from drawdowns

### 4. Correlation Analysis

#### Strategy-to-Portfolio Correlation
**Definition:** How closely strategy performance moves with overall portfolio performance.

**Calculation:**
```python
def calculate_strategy_portfolio_correlation(
    strategy_returns: List[float],
    portfolio_returns: List[float],
    timeframe: str = '1D'
) -> CorrelationMetrics:
    """
    Calculate correlation between strategy and portfolio performance.
    
    Returns correlation coefficient, p-value, and regime analysis.
    """
    # Ensure equal length arrays
    min_length = min(len(strategy_returns), len(portfolio_returns))
    strategy_returns = strategy_returns[-min_length:]
    portfolio_returns = portfolio_returns[-min_length:]
    
    # Calculate Pearson correlation
    correlation, p_value = scipy.stats.pearsonr(strategy_returns, portfolio_returns)
    
    # Rolling correlation for stability analysis
    window_size = min(30, len(strategy_returns) // 4)  # 30-day window or 1/4 of data
    rolling_correlations = []
    
    for i in range(window_size, len(strategy_returns)):
        window_strategy = strategy_returns[i-window_size:i]
        window_portfolio = portfolio_returns[i-window_size:i]
        window_corr, _ = scipy.stats.pearsonr(window_strategy, window_portfolio)
        rolling_correlations.append(window_corr)
    
    return CorrelationMetrics(
        correlation=correlation,
        p_value=p_value,
        stability_score=1.0 - statistics.stdev(rolling_correlations),  # Lower stdev = more stable
        average_rolling_correlation=statistics.mean(rolling_correlations),
        correlation_trend=calculate_correlation_trend(rolling_correlations)
    )
```

#### Cross-Strategy Correlation Matrix
**Purpose:** Identify redundant or diversifying strategies.

**Implementation:**
```python
def build_strategy_correlation_matrix(
    strategies: List[str],
    strategy_returns: Dict[str, List[float]]
) -> pd.DataFrame:
    """
    Build correlation matrix between all strategy pairs.
    """
    correlation_matrix = pd.DataFrame(index=strategies, columns=strategies)
    
    for strategy1 in strategies:
        for strategy2 in strategies:
            if strategy1 == strategy2:
                correlation_matrix.loc[strategy1, strategy2] = 1.0
            else:
                returns1 = strategy_returns[strategy1]
                returns2 = strategy_returns[strategy2]
                
                # Align time series
                aligned_returns = align_return_series(returns1, returns2)
                correlation, _ = scipy.stats.pearsonr(
                    aligned_returns[strategy1], 
                    aligned_returns[strategy2]
                )
                correlation_matrix.loc[strategy1, strategy2] = correlation
    
    return correlation_matrix.astype(float)
```

#### Ticker Cross-Correlation
**Definition:** Correlation between tickers within the same strategy to assess diversification.

**Key Metrics:**
- **Average Cross-Correlation:** Mean correlation between all ticker pairs in strategy
- **Highest Correlation Pair:** Most correlated tickers (potential redundancy)
- **Diversification Score:** `1 - average_cross_correlation` (higher = better diversified)

### 5. Regime Dependency Analysis

#### Market Regime Detection
**Purpose:** Understand how strategy performance varies with market conditions.

**Regime Categories:**
- **Bull Market:** S&P 500 > 20-day MA, VIX < 20
- **Bear Market:** S&P 500 < 20-day MA, VIX > 30  
- **Volatile Market:** VIX > 25 regardless of price trend
- **Low Vol Market:** VIX < 15 regardless of price trend

```python
def analyze_regime_performance(
    signals: List[Signal],
    market_regime_data: List[MarketRegime]
) -> RegimeAnalysis:
    """
    Analyze strategy performance across different market regimes.
    """
    regime_performance = {}
    
    for regime_type in ['bull', 'bear', 'volatile', 'low_vol']:
        regime_signals = []
        regime_returns = []
        
        for signal in signals:
            regime = get_regime_at_date(market_regime_data, signal.timestamp)
            if regime.type == regime_type:
                signal_return = calculate_signal_return(signal)
                regime_signals.append(signal)
                regime_returns.append(signal_return)
        
        if regime_returns:
            regime_performance[regime_type] = {
                'signal_count': len(regime_signals),
                'win_rate': calculate_win_rate(regime_signals),
                'avg_return': statistics.mean(regime_returns),
                'return_volatility': statistics.stdev(regime_returns) if len(regime_returns) > 1 else 0,
                'sharpe_ratio': calculate_sharpe_ratio(regime_returns),
                'max_drawdown': calculate_max_drawdown(regime_returns)
            }
    
    return RegimeAnalysis(
        regime_performance=regime_performance,
        best_regime=max(regime_performance.items(), key=lambda x: x[1]['sharpe_ratio'])[0],
        worst_regime=min(regime_performance.items(), key=lambda x: x[1]['sharpe_ratio'])[0],
        regime_consistency=calculate_regime_consistency(regime_performance)
    )
```

## Configurable Timeframes

### Timeframe Definitions

| Timeframe | Signal Window | Return Measurement | Use Case |
|-----------|---------------|-------------------|-----------|
| **1D** | 24 hours | Next-day close | Day trading evaluation |
| **1W** | 7 days | Weekly close | Swing trading analysis |
| **1M** | 30 days | Monthly close | Position trading assessment |
| **3M** | 90 days | Quarterly close | Long-term strategy validation |

### Dynamic Timeframe Selection
```python
class TimeframeManager:
    def __init__(self):
        self.timeframes = {
            '1D': timedelta(days=1),
            '1W': timedelta(days=7), 
            '1M': timedelta(days=30),
            '3M': timedelta(days=90)
        }
    
    def calculate_all_timeframes(
        self, 
        strategy_id: str, 
        ticker_symbol: str
    ) -> Dict[str, AttributionMetrics]:
        """
        Calculate attribution metrics across all timeframes for a strategy-ticker pair.
        """
        results = {}
        signals = self.get_signals(strategy_id, ticker_symbol)
        
        for timeframe_name, timeframe_delta in self.timeframes.items():
            results[timeframe_name] = AttributionMetrics(
                win_rate=self.calculate_win_rate(signals, timeframe_delta),
                avg_return=self.calculate_avg_return(signals, timeframe_delta),
                max_gain=self.calculate_max_gain(signals, timeframe_delta),
                max_loss=self.calculate_max_loss(signals, timeframe_delta),
                sharpe_ratio=self.calculate_sharpe_ratio(signals, timeframe_delta),
                max_drawdown=self.calculate_max_drawdown(signals, timeframe_delta),
                correlation_to_portfolio=self.calculate_portfolio_correlation(signals, timeframe_delta)
            )
        
        return results
```

## Performance Requirements

### Calculation Frequency
- **Real-time Updates:** Every 5 seconds during market hours
- **Historical Recalculation:** Nightly batch process
- **On-demand Calculation:** Modal opens, timeframe changes

### Scalability Targets
- **676 Tickers × 5 Strategies = 3,380 Combinations**
- **4 Timeframes × 3,380 = 13,520 Metric Sets**
- **Target Response Time:** <500ms for modal load
- **Memory Usage:** <100MB for full attribution cache

This comprehensive attribution system provides the analytical depth needed to optimize trading strategies while maintaining real-time responsiveness for active portfolio management.
